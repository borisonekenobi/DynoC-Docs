Variables and Scope

Variables in DynoC are used to store data that can be read or modified during program execution.

Each variable has a name, a type, and a value.

Since DynoC is statically typed, you must declare a variable’s type before assigning a value.

Declaring Variables

To declare a variable, specify its type followed by a name and an optional initial value:

 count;
 pi = 3.1415;
 active = ;

If you declare a variable without assigning a value, it contains that type's default value.

Assigning Values

Use the = operator to assign or update a variable’s value.

 score = 100;
score = 150; // updated

The type of a variable cannot change after declaration.

 x = 5;
x = 10;           // valid
x = Hello Mom!; // invalid (type mismatch)

Constants

DynoC does not yet include a const keyword, but variables that are never reassigned can be treated as constants by convention.

 GRAVITY = 9.81; // treat as constant

Future versions may introduce explicit constant declarations.

Scope

Scope determines where a variable can be accessed within a program.

DynoC uses block scope, meaning variables exist only inside the {} block where they are defined.

 example()
{
     a = 10;

     (a > 5)
    {
         b = 20;
        (a); // OK
        (b); // OK
    }

    (a); // OK
    (b); // Error: b is out of scope
}

When the inner block ends, b is destroyed and cannot be used outside it.

Global Variables

Variables declared outside any function are global.

They can be accessed from anywhere in the program unless shadowed by a local variable of the same name.

 version = 3;

 show_version()
{
    (version); // accessible here
}

Global variables remain in memory for the entire program lifetime.

Variable Shadowing

Currently, variable shadowing is not allowed, and the compiler will throw an error if it's tried.

 value = 10;

 test()
{
   value = 5; // Error: 'value' already exists
  (value);
}
