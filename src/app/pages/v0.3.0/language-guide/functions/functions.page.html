Functions

Functions in DynoC are reusable blocks of code that perform a specific task.

They help organize code, reduce repetition, and improve readability.

Each function can take input values (called parameters) and optionally return a result.

Defining a Function

Functions are defined with their type followed by the function name, parameters (if any), and a block of code in {}.

 greet()
{
    (Hello from DynoC!);
}

Function definitions can appear anywhere in the source file, outside of other functions, but they must be defined before they are called.

Calling a Function

To call a function, use its name followed by parentheses ().

greet();

If the function has parameters, pass values inside the parentheses.

 greet_user( name)
{
    (Hello + name + !);
}

greet_user(Boris);

Parameters

Parameters act as placeholders for input values.

You define them inside the parentheses after the function name.

 add( a,  b)
{
     sum = a + b;
    (sum);
}

You can pass expressions or variables as arguments:

 x = 3;
 y = 7;
add(x, y); // prints 10

Parameters are local to the function and are destroyed when the function ends.

Return Values

Functions can return a value using the return keyword.

 multiply( a,  b)
{
     a * b;
}

 result = multiply(4, 5);
(result); // 20

Recursion

A function can call itself, this is called recursion.

Recursion is useful for tasks that can be defined in terms of smaller subproblems.

 factorial( n)
{
     (n <= 1)  1;
     n * factorial(n - 1);
}

(factorial(5)); // 120

Function Overloading

DynoC supports function overloading.

You can define multiple functions with the same name as long as their parameter types or counts are different.

 add( a,  b)
{
     a + b;
}

 add( a,  b)
{
     a + b;
}

 add( a,  b,  c)
{
     a + b + c;
}

(add(2, 3));     // calls add(int, int)
(add(2.5, 3.1)); // calls add(float, float)
(add(1, 2, 3));  // calls add(int, int, int)

The correct function version is chosen based on the number and types of arguments passed.
