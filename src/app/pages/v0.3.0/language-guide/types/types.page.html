Types

DynoC is currently statically typed, meaning each variable and expression has a fixed type at compile time.

You must declare a variableâ€™s type before assigning a value, and the type cannot change later.

Types

DynoC provides several primitive types:

int, float, bool, char, string, array

Integer (int)

The int type stores whole numbers, both positive and negative.

 a = 10;
 b = -3;
 sum = a + b; // 7

Division between two integers truncates toward zero:

 x = 7 / 2; // 3

Floating Point (float)

The float type stores decimal numbers.

 x = 3.14;
 y = -0.5;
 z = x + y; // 2.64

Operations between int and float are implicitly casted:

 a = 3;
 b = 2.5;
 result = a * b; // 7.5

Boolean (bool)

The bool type represents a logical value: true or false.

 is_ready = ;
 passed = (score >= 50);

Booleans can be used directly in control flow statements:

 (is_ready)
{
    print_line(Ready to go!);
}

Character (char)

The char type stores a single ASCII or Extended-ASCII (values between 0-255) character in single quotes. 

 c = A;

A char can be converted to an int to get its numeric value, or vice versa using implicit casting:

 code = A; // 65
 letter = 66 // 'B'

String (string)

The string type stores a sequence of characters in double quotes.

 name = DynoC;

You can concatenate strings using the + operator:

 msg = Hello,  + name + !;

Arrays ([])

Arrays hold an ordered collection of elements of the same type.

[] nums = [1, 2, 3, 4];

Arrays can be defined for any type:

[] names = [Alice, Bob, Charlie];

You can access and modify elements using indices:

nums[1] = 10;        // nums = [1, 10, 3, 4]
print_line(nums[0]); // 1

The length of an array is stored in the length property:

print_line(There is , nums.length,  numbers in the nums array.);

Type Casting

DynoC currently only supports implicit type casting. (this will change in the future!)

 a = 3;   // a = 3.0
 b = a / 2; // b = 1

Casting does not modify the original variable unless reassigned.
